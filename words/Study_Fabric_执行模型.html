1.默认执行模型

说明：Fabric默认是单一的，串行执行

例子

Bash
#代码部分
    from fabric.api import env, run

    env.hosts = ['host1', 'host2']
    def mytask():
        run('ls /var/www')
    
    #命令行执行
    fab taskA taskB

    #执行过程
    taskA executed on host1
    taskA executed on host2
    taskB executed on host1
    taskB executed on host2

2.定义主机列表

Bash
host定义形式: username@hostname:port
        例子:admin@willdx.com:22

    注释:也支持ipv6
        例子:user@[2001:db8::1]:1222

    设定主机列表:
        (1)在模块级别定义
        #执行fab mytask1 和 fab mytask2都将会直接在host1和host2上执行
        from fabric.api import env, run
        env.hosts = ['host1', 'host2']
        def mytask1():
            run('ls /var/test1')
        def mytask2():
            run('ls /var/test2')

        (2)在任务级别定义
        #执行fab set_host1 mytask1 会在host1上执行
        #执行fab set_host2 mytask2 会在host2上执行
        from fabric.api import env, run
        def set_host1():
            env.hosts = ['host1']
        def set_host1():
            env.hosts = ['host2']
        def mytask1():
            run('ls /var/test1')
        def mytask2():
            run('ls /var/test2')

        (3)同时设定模块和任务级别
        #可以同时用以下2种定义方式,他们的值互相重写而不是合并
        from fabric.api import env, run
        env.hosts = ['host1', 'host2']
        def set_host1():
            env.hosts = ['host1']
        def mytask1():
            run('ls /var/test1')
        #若使用fab mytask1 那么会在host1和host2上执行
        #若使用fab set_host1 mytask1 只会字在host1上执行

        (4)在命令行使用`-H HOSTS, --hosts=HOSTS`全局环境定义hosts
        fab -H host1,host2 mytask
        注：
        在命令行使用`-R ROLES, --roles=ROLES`在全局环境下定义角色
        在命令行用`-x HOSTS, --exclude-hosts=HOSTS` 定义排除的host
        (PS:更多fab设置请参见: fab --help)

    添加主机列表
        from fabric.api import env, run
        env.hosts.extend(['host3', 'host4'])
        def mytask():
            run('ls /var/www')
        run: fab -H host1,host2 mytask, env.hosts 
        executed:将会在['host1', 'host2', 'host3', 'host4']上执行 
        (PS: env.hosts只是一个列表对象,所以也可以使用append方法)

    确保任务只在指定的host中执行
        #This will override any other host list and ensure mytask always runs on just those two hosts.
        方法1:fab mytask:hosts="host1;host2"
        方法2:装饰器
        my_hosts = ('host1', 'host2')
        @hosts(my_hosts)
        def mytask():

     主机列表合并
        from fabric.api import env, hosts, roles, run
        env.roledefs = {'role1': ['b', 'c']}
        @hosts('a', 'b')
        @roles('role1')
        def mytask():
            run('ls /var/www')
        #将会执行['a', 'b', 'c']

    主机列表重复数据删除
        默认情况下Fabric会将主机列表去重,env.dedupe_hosts = True
        如果要允许重复需要:env.dedupe_hosts = False 关闭去重功能
    
    排除特定主机
        fab -R myrole -x host2,host5 mytask
        fab mytask:roles=myrole,exclude_hosts="host2;host5"
        注：装饰器设定的主机列表能够排除
        一个任务功能，装饰着@hosts(host1,host2)exclude_hosts=host2将只能运行在host1

3.定义角色

roledefs可以用于将主机分组
#在env.roledefs中定义了role和host的对应关系,其中host存在于env.hosts,其中role存在于env.roles

Bash
from fabric.api import env
    #定义roles
    env.roledefs = {
    'web': ['www1', 'www2', 'www3'],
    'dns': ['ns1', 'ns2']
    }
    #添加一个role
    env.roledefs['webservers'] = ['www1', 'www2', 'www3']
    #roledefs是一个字典,key用role名称表示,value既可以定义为主机的列表，也可以定义为一个字典(PS:主机列表为`hosts键`的值) 
    #例子
    env.roledefs = {
    'web': {
        'hosts': ['www1', 'www2', 'www3'],
        'foo': 'bar'
    },
    'dns': {
        'hosts': ['ns1', 'ns2'],
        'foo': 'baz'
    }}

4.执行任务execute

Bash
1.命令行下 fab  task
    2.execute(函数名称)

    from fabric.api import run, roles

    env.roledefs = {
        'db': ['db1', 'db2'],
        'web': ['web1', 'web2', 'web3'],
    }
    
    @roles('db')
    def migrate():
        # Database stuff here.
        pass
    
    @roles('web')
    def update():
        # Code updates here.
        pass

    (1)fab migrate update
    (2)在上述脚本中添加如下代码：
    from fabric.api import run, roles, execute
    def deploy():
        execute(migrate)
        execute(update)
    然后在命令行:fab deploy

5.获得结果

Bash
from fabric.api import task, execute, run, runs_once

    @task
    def workhorse():
        return run("get my infos")
    
    @task
    @runs_once
    def go():
        results = execute(workhorse)
        print results

Raw

但是，如果env.warn_only设置为真，在失败的时候-有，比方说，在设置上下文管理器-布会发出警告信息，但继续执行

Raw
