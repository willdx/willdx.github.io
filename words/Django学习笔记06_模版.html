1.模版的执行

　　模版的创建过程，对于模版，其实就是读取模版（其中嵌套着模版标签），然后将 Model 中获取的数据插入到模版中，最后将信息返回给用户。

Vserion01:返回一个带HTML标签的字符串到浏览器

Bash
def current_datetime(request):
now = datetime.datetime.now()
html = "<html><body>It is now %s.</body></html>" % now
return HttpResponse(html)

Vsersion02:使用模版系统，将Context作为参数传递给template.Template最终返回到浏览器

Bash
from django import template
t = template.Template('My name is {{ name }}.')
c = template.Context({'name': 'Adrian'})
print t.render(c)

Vserion03:将HTML文件写入文件，读取文件到模版中，使用render方法将Context作为参数传递给template.Template最终返回到浏览器

Bash
import datetime
from django import template
import DjangoDemo.settings
 
now = datetime.datetime.now()
fp = open(settings.BASE_DIR+'/templates/Home/Index.html')
t = template.Template(fp.read())
fp.close()
html = t.render(template.Context({'current_date': now}))
return HttpResponse(html)

Vsersion04:rendertoresponse方法，简化读取HTML文件模版过程，一条命令搞定

Bash
from django.http.response import HttpResponse
from django.shortcuts import render,render_to_response
from django.template import RequestContext

return render_to_response('user.html',RequestContext(request,{'info':user_info}))
等同于：
return render_to_response('user.html',user_info,context_instance=RequestContext(request))

注意:解决跨站请求伪造报错问题,RequestContext

2.7 模板中获取request信息

  • 当前网址,用户等信息

如果不是在 views.py 中用的 render 函数，是 rendertoresponse 的话，需要将 request 加入到 上下文渲染器; Django 1.11 中新建的项目已经默认加上了;

Django 1.8 及以后 修改 settings.py;

Python
TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                ...
                'django.template.context_processors.request',
                ...
            ],
        },
    },
]

Django 1.7 及以前 修改 settings.py

Python
TEMPLATE_CONTEXT_PROCESSORS = (
    "django.contrib.auth.context_processors.auth",
    "django.core.context_processors.debug",
    "django.core.context_processors.i18n",
    "django.core.context_processors.media",
    "django.core.context_processors.static",
    "django.core.context_processors.tz",
    "django.contrib.messages.context_processors.messages",
    "django.core.context_processors.request",
)

3.通过simple_tag自定义filter

Python
a、在app中创建templatetags文件夹

b、创建任意 .py 文件，如：xx.py

#!/usr/bin/env python
#coding:utf-8
from django import template
from django.utils.safestring import mark_safe
from django.template.base import resolve_variable, Node, TemplateSyntaxError
 
register = template.Library()
 
@register.simple_tag
def my_simple_time(v1,v2,v3):
    return  v1 + v2 + v3
 
@register.simple_tag
def my_input(id,arg):
    result = "<input type='text' id='%s' class='%s' />" %(id,arg,)
    return mark_safe(result)

5.模板的位置

模板一般放在app下的templates中，Django会自动去这个文件夹中找; 在view.py文件中置顶模板路径的时候, 默认我们可以直接写:

Python
def home(request):
    user = {"id": 1}
    return render(request, 'home.html', {"user": user})

但当有多个APP, 且每个APP的templates模块中都有相同的home.html文件的时候, 可能会查询到其他APP的同名文件;

避免这个问题的方法是:

  • 第一步: 在每个APP的templates文件夹中建一个和app同名的目录, 将app相关的模板文件都放入到这个目录中;

  • 第二步: 在view.py文件中指定模板文件的时候加上APP的名称即可; 如下:

Python
def home(request):
    user = {"id": 1}
    return render(request, 'accouting_app/home.html', {"user": user})
